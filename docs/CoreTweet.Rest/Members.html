 Create (params System.Linq.Expressions.Expression<Func<string, object>>[]) : CoreTweet.List

Add a member to a list. The authenticated user must own the list to be able to add members to it. Note that lists can't have more than 500 members.

Note: Either a list_id or a slug is required. If providing a list_slug, an owner_screen_name or owner_id is also required.

Avaliable parameters:

long list_id (required) : The numerical id of the list.

string slug (required) : You can identify a list by its slug instead of its numerical id. If you decide to do so, note that you'll also have to specify the list owner using the owner_id or owner_screen_name parameters.

long user_id (required) : The ID of the user for whom to return results for. Helpful for disambiguating when a valid user ID is also a valid screen name.

string screen_name (required) : The screen name of the user for whom to return results for. Helpful for disambiguating when a valid screen name is also a user ID.

string owner_screen_name (optional) : The screen name of the user who owns the list being requested by a slug.

long owner_id (optional) : The user ID of the user who owns the list being requested by a slug.
CreateAll (params System.Linq.Expressions.Expression<Func<string, object>>[]) : CoreTweet.List

Adds multiple members to a list, by specifying a comma-separated list of member ids or screen names. The authenticated user must own the list to be able to add members to it. Note that lists can't have more than 500 members, and you are limited to adding up to 100 members to a list at a time with this method.

Please note that there can be issues with lists that rapidly remove and add memberships. Take care when using these methods such that you are not too rapidly switching between removals and adds on the same list.

Note: Either a list_id or a slug is required. If providing a list_slug, an owner_screen_name or owner_id is also required.

Avaliable parameters:

long list_id (required) : The numerical id of the list.

string slug (required) : You can identify a list by its slug instead of its numerical id. If you decide to do so, note that you'll also have to specify the list owner using the owner_id or owner_screen_name parameters.

string user_id (optional) : A comma separated list of user IDs, up to 100 are allowed in a single request.

string screen_name (optional) : A comma separated list of screen names, up to 100 are allowed in a single request.

string owner_screen_name (optional) : The screen name of the user who owns the list being requested by a slug.

long owner_id (optional) : The user ID of the user who owns the list being requested by a slug.

This (params System.Linq.Expressions.Expression<Func<string, object>>[]) : CoreTweet.Cursored<CoreTweet.User>

Returns the members of the specified list. Private list members will only be shown if the authenticated user owns the specified list.

Note: Either a list_id or a slug is required. If providing a list_slug, an owner_screen_name or owner_id is also required.

The response from the API will include a previous_cursor and next_cursor to allow paging back and forth. See Using cursors to navigate collections for more information.

https://dev.twitter.com/docs/misc/cursoring

Avaliable parameters:

long list_id (required) : The numerical id of the list.

string slug (required) : You can identify a list by its slug instead of its numerical id. If you decide to do so, note that you'll also have to specify the list owner using the owner_id or owner_screen_name parameters.

string owner_sereen_name (optional) : The screen name of the user who owns the list being requested by a slug.

long owner_id (optional) : The user ID of the user who owns the list being requested by a slug.

long cursor (semi-optional) : Causes the collection of list members to be broken into "pages" of somewhat consistent size. If no cursor is provided, a value of -1 will be assumed, which is the first "page".
