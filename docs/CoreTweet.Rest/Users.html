Lookup (params System.Linq.Expressions.Expression<Func<string, object>>[]) : IEnumerable<CoreTweet.User>

Returns fully-hydrated user objects for up to 100 users per request, as specified by comma-separated values passed to the user_id and/or screen_name parameters.

This method is especially useful when used in conjunction with collections of user IDs returned from GET friends/ids and GET followers/ids.

GET users/show is used to retrieve a single user object.

Avaliable parameters:

string user_id (optional) : A comma separated list of user IDs, up to 100 are allowed in a single request. You are strongly encouraged to use a POST for larger requests.

string screen_name (optional) : A comma separated list of screen names, up to 100 are allowed in a single request. You are strongly encouraged to use a POST for larger (up to 100 screen names) requests.

bool include_entities (optional) : The entities node that may appear within embedded statuses will be disincluded when set to false.
ProfileBanner (params System.Linq.Expressions.Expression<Func<string, object>>[]) : CoreTweet.Size

Returns the size of the specified user's profile banner. If the user has not uploaded a profile banner, a HTTP 404 will be served instead. This method can be used instead of string manipulation on the profile_banner_url returned in user objects as described in User Profile Images and Banners.

Note: Always specify either an user_id or screen_name when requesting this method.

Avaliable parameters:

long id (optional) : The ID of the user for whom to return results for. Helpful for disambiguating when a valid user ID is also a valid screen name.

string screen_name (optional) : The screen name of the user for whom to return results for. Helpful for disambiguating when a valid screen name is also a user ID.
ReportSpam (params System.Linq.Expressions.Expression<Func<string, object>>[]) : CoreTweet.User

Report the specified user as a spam account to Twitter. Additionally performs the equivalent of POST blocks/create on behalf of the authenticated user.

Note: One of these parameters must be provided.

Avaliable parameters:

string screen_name (optional) : The ID or screen_name of the user you want to report as a spammer. Helpful for disambiguating when a valid screen name is also a user ID.

long user_id (optional) : The ID of the user you want to report as a spammer. Helpful for disambiguating when a valid user ID is also a valid screen name.
Search (params System.Linq.Expressions.Expression<Func<string, object>>[]) : IEnumerable<CoreTweet.User>

Provides a simple, relevance-based search interface to public user accounts on Twitter. Try querying by topical interest, full name, company name, location, or other criteria. Exact match searches are not supported.

Only the first 1,000 matching results are available.

Avaliable parameters:

string q (required) : The search query to run against people search.

int page (optional) : Specifies the page of results to retrieve.

int count (optional) : The number of potential user results to retrieve per page. This value has a maximum of 20.

bool include_entities (optional) : The entities node will be disincluded from embedded tweet objects when set to false.
Show (params System.Linq.Expressions.Expression<Func<string, object>>[]) : CoreTweet.User

Returns a variety of information about the user specified by the required user_id or screen_name parameter. The author's most recent Tweet will be returned inline when possible.

GET users/lookup is used to retrieve a bulk collection of user objects.

Avaliable parameters:

long user_id (required) : The ID of the user for whom to return results for. Either an id or screen_name is required for this method.

string screen_name (required) : The screen name of the user for whom to return results for. Either a id or screen_name is required for this method.

bool include_entities (optional) : The entities node will be disincluded when set to false.
SuggestedMembers (params System.Linq.Expressions.Expression<Func<string, object>>[]) : IEnumerable<CoreTweet.User>

Access the users in a given category of the Twitter suggested user list and return their most recent status if they are not a protected user.

Avaliable parameters:

string slug (required) : The short name of list or a category
Suggestions (params System.Linq.Expressions.Expression<Func<string, object>>[]) : IEnumerable<CoreTweet.Category>

Access to Twitter's suggested user list. This returns the list of suggested user categories. The category can be used in GET users/suggestions/:slug to get the users in that category.

Avaliable parameters:

string lang (optional) : Restricts the suggested categories to the requested language. The language must be specified by the appropriate two letter ISO 639-1 representation. Currently supported languages are provided by the GET help/languages API request. Unsupported language codes will receive English (en) results. If you use lang in this request, ensure you also include it when requesting the GET users/suggestions/:slug list.
