 Create (params System.Linq.Expressions.Expression<Func<string, object>>[]) : CoreTweet.User

Allows the authenticating users to follow the user specified in the ID parameter.

Returns the befriended user in the requested format when successful. Returns a string describing the failure condition when unsuccessful. If you are already friends with the user a HTTP 403 may be returned, though for performance reasons you may get a 200 OK message even if the friendship already exists.

Actions taken in this method are asynchronous and changes will be eventually consistent.

Note: Providing either screen_name or user_id is required.

Avaliable parameters:

string screen_name (optional) : The screen name of the user for whom to befriend.

long user_id (optional) : The ID of the user for whom to befriend.

bool follow (optional) : Enable notifications for the target user.

Incoming (params System.Linq.Expressions.Expression<Func<string, object>>[]) : CoreTweet.Cursored<long>

Returns a collection of numeric IDs for every user who has a pending request to follow the authenticating user.

Avaliable parameters:

long cursor (semi-optional) : Causes the list of connections to be broken into pages of no more than 5000 IDs at a time. The number of IDs returned is not guaranteed to be 5000 as suspended users are filtered out after connections are queried. If no cursor is provided, a value of -1 will be assumed, which is the first "page." The response from the API will include a previous_cursor and next_cursor to allow paging back and forth. See Using cursors to navigate collections for more information.
Lookup (params System.Linq.Expressions.Expression<Func<string, object>>[]) : IEnumerable<CoreTweet.Friendship>

Returns the relationships of the authenticating user to the comma-separated list of up to 100 screen_names or user_ids provided. Values for connections can be: following, following_requested, followed_by, none.

Avaliable parameters:

string screen_name (optional) : A comma separated list of screen names, up to 100 are allowed in a single request.

string user_id (ooptional) : A comma separated list of user IDs, up to 100 are allowed in a single request.
NoRetweetsIds (params System.Linq.Expressions.Expression<Func<string, object>>[]) : IEnumerable<long>

Returns a collection of user_ids that the currently authenticated user does not want to receive retweets from.

Use POST friendships/update to set the "no retweets" status for a given user account on behalf of the current user.

Avaliable parameters: Nothing.
